# 系统说明

## 一、项目定位
本项目是一个基于 RBAC（角色访问控制）与数据权限控制的后台管理系统，用于支撑企业内部用户、组织、角色与权限的统一管理。

## 二、技术架构

### 1. 后端技术栈
- 框架：Spring Boot 3
- JDK：JDK 21
- 数据库：PostgreSQL（pgsql）

#### 后端职责
- 提供用户、组织、角色、权限等核心业务接口。
- 提供认证鉴权能力（登录、JWT、权限校验）。
- 提供菜单权限与数据权限的统一控制。
- 提供日志与安全能力（登录日志、操作日志、敏感操作校验）。

### 2. 前端技术栈
- 框架：Vue 3

#### 前端职责
- 提供系统管理界面（用户、组织、角色、权限配置）。
- 登录后拉取权限并动态生成菜单。
- 根据按钮权限标识控制页面操作项显隐。
- 与后端接口联动完成增删改查及权限化展示。

## 三、前后端协作方式
- 前端通过登录接口获取 JWT。
- 后续请求携带 JWT 访问后端受保护接口。
- 后端按角色与权限规则返回可访问资源及数据范围。
- 前端依据权限数据动态渲染菜单、页面按钮与功能入口。

## 四、数据与安全说明
- 数据落库使用 PostgreSQL。
- 用户密码使用 BCrypt 加密存储。
- 核心接口执行权限注解校验，防止越权调用。
- 对关键操作保留审计日志，满足问题追踪与安全审计需求。
