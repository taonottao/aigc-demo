# 权限管理系统 PRD

## 一、项目目标
构建一个通用的后台权限管理系统，覆盖用户、组织、角色、权限与系统安全支撑能力，支持企业内部多组织、多角色、多数据范围的管理场景。

## 二、功能需求

### 1. 用户管理

#### 1.1 基础操作
- 支持用户增删改查。
- 支持用户启用/禁用。
- 支持批量导入与批量导出。

#### 1.2 用户信息字段
- 用户名（唯一）。
- 加密密码。
- 姓名。
- 手机号。
- 邮箱。
- 头像。

#### 1.3 关联配置
- 归属组织：单归属，简化设计。
- 分配角色：支持一个用户分配多个角色。

#### 1.4 自助与管理能力
- 个人中心：用户可修改个人资料与密码。
- 管理员可重置用户密码。

### 2. 组织管理（树形架构）

#### 2.1 组织维护
- 支持组织增删改查。
- 支持拖拽排序。
- 支持设置父级组织。
- 支持无限层级组织结构。

#### 2.2 组织信息字段
- 名称。
- 编码。
- 负责人。
- 描述。

#### 2.3 成员视图
- 在组织节点下可直接查看成员。
- 在组织节点下可直接添加成员。
- 在组织节点下可直接移除成员。
- 组织与成员为双向关联。

### 3. 角色管理（权限载体）

#### 3.1 角色配置
- 支持创建角色（如超级管理员、部门主管）。
- 支持设置角色状态（启用/禁用）。

#### 3.2 权限绑定
- 菜单权限：勾选菜单权限，控制前端菜单/按钮可见性。
- 数据权限：按业务模块分配数据权限规则（如用户管理、订单模块）。

#### 3.3 用户分配
- 支持将角色赋予用户。
- 支持多角色叠加。

### 4. 权限管理

#### 4.1 菜单权限
- 菜单树管理：名称、路由、图标、父级、排序。
- 按钮级权限标识：如 `user:delete`。
- 前端通过指令/组件控制按钮显隐。

#### 4.2 数据权限（行级）
- 预设规则：
  - 全部数据
  - 本部门及子部门
  - 仅本部门
  - 仅本人
- 支持按模块配置规则：
  - 示例：订单模块配置为“本部门”
  - 示例：报表模块配置为“全部”
- 实现提示：后端查询时动态拼接 SQL 条件（如 MyBatis 拦截器）。

### 5. 系统支撑

#### 5.1 认证登录
- 用户名密码登录。
- 验证码防刷。
- JWT 会话管理。

#### 5.2 权限校验
- 前端：登录后拉取权限，动态生成菜单并控制按钮显隐。
- 后端：接口层注解校验（如 `@PreAuthorize("hasRole('ADMIN')")`）。

#### 5.3 基础日志
- 登录日志。
- 关键操作日志（用户、时间、操作内容）。

#### 5.4 安全基础
- 密码采用 BCrypt 加密。
- 敏感操作二次验证（如删除操作）。

## 三、非功能要求（建议）
- 可维护性：模块边界清晰，支持后续扩展业务模块。
- 安全性：权限校验前后端双保险，防止越权访问。
- 可审计性：关键操作全链路留痕。

## 四、阶段性验收要点
- 用户、组织、角色、菜单、数据权限核心流程可用。
- 前端菜单与按钮显隐与权限配置一致。
- 后端数据权限生效且可按模块配置。
- 登录、鉴权、日志、安全策略落地并可验证。
